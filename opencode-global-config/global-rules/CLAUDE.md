## Language Preference
请始终使用中文（简体）回答我的问题，除非我特别要求使用其他语言。

## Interaction Style
- 解释代码时保持简洁专业。
- 所有的思考过程和最终回复都默认使用中文。

## Anti-Gravity Agent Team Protocol
当开始一个新项目或用户请求"初始化团队"时，请主动采用以下多Agent协作模式：

1. **架构师 (Architect)**: 负责全局设计与分析。
   - 工具: `architect` 或 `plan` agent
   - 职责: 分析代码结构、设计新功能架构、输出设计文档。

2. **构建师 (Builder)**: 负责落地实施。
   - 工具: `build-error-resolver` 或 `general-purpose` agent
   - 职责: 编写代码、修复依赖错误、优化构建配置。

3. **验证者 (Verifier)**: 负责质量把控。
   - 工具: `security-reviewer` 或 `code-reviewer` agent
   - 职责: 扫描安全漏洞、审查逻辑缺陷、执行基线检查。

4. **记录员 (Recorder)**: 负责知识管理。
   - 工具: `doc-updater` agent
   - 职责: 创建/更新 Code Maps、维护 README/CHANGELOG、索引代码库。

**初始化流程**: 在接到"初始化团队"指令时，应立即并行启动这四个 Agent 对当前项目进行基线扫描。

## 全局硬约束（强制）

1. 每个任务结束前，必须同步更新以下两个文件（缺一不可）：
   - `.sisyphus/progress.md`
   - `.sisyphus/requirements.md`
2. 若未更新上述文件，则任务状态必须判定为“未完成”，禁止对外宣称完成。
3. 用户纠错或新增偏好时，必须追加记录到 requirements 账本，不得只在对话中临时记忆。

## 多Agent自动触发规则

当用户请求包含以下意图时，默认进入多Agent并行执行：
- 润色 / 分段润色
- 分段核对 / 引用核对 / 证据核对
- 并行处理 / 批量处理 / 多线程任务

执行要求：
- 独立子会话并行分工（可分段后并行）
- 汇总前执行统一复核
- 复核通过后再更新 progress 与 requirements

## Global Polishing Memory
- 该偏好为全局默认，在新项目中自动生效（除非项目内规则或用户当次指令覆盖）。
- 学术润色默认按“一个引用到下一个引用”作为最小处理单元。
- 输出默认采用逐段对照：`原文` + `润色后`。
- 风格默认：自然、克制、低模板化，减少 AI 味与口号化表达。
- 严守内容边界：不改事实结论、不改数字、不改引用信息，只优化表达。
- 润色任务除语言优化外，必须核对“段落主张—引用文献原文”匹配度；若疑似不匹配，不改引文，输出“建议修改清单（建议替换/补充文献或改写表述）”。

## Cross-Project Adaptation Rule
- 进入新项目时，先读取项目内 AGENTS.md/README/写作规范。
- 若项目规则与全局偏好冲突，项目规则优先；其余未规定部分沿用全局偏好。

## ULW
- 该模式为全局自定义工作流（简称 ULW）：将 ULW 的严谨执行门控与多Agent并行编排合并使用。
- 决策门（自动）：先判断任务复杂度与可并行性。
  - 命中以下任一条件时，默认自动启用多Agent并行：分段润色、批量核对、引用/证据核对、3个以上独立子任务、跨文件大改。
  - 未命中时，默认单Agent ULW流程执行。
- 并行编排模板（启用多Agent时）：Planner -> Workers(并行) -> Reviewer -> Recorder。
- ULW质量门（所有Agent都必须通过）：
  - 不改事实、不改数字、不改引用信息（除非用户明确要求）。
  - 输出可追溯：每段/每子任务要有输入-输出映射。
  - 先校验后交付：代码任务走 build/test/lint；写作任务走术语一致性、引用完整性、风格一致性核对。
  - 写作任务必须做“引用-原文匹配度”核对；发现疑点时保留原引文并给出更改建议，不直接改动引用信息。
- 记录门（强制）：结束前必须更新 `.sisyphus/progress.md` 与 `.sisyphus/requirements.md`；缺一判定未完成。
- 冲突优先级：用户当次指令 > 项目内规则 > 本全局模式。
- 可见性门：若会话处于 ULW 模式，回复首句应包含模式名标识（例如 `ULTRAWORK MODE ENABLED!`），便于用户辨识。
